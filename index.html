03 / Foundations

The CIA Triad

Cybersecurity is the practice of protecting systems, networks, and data from unauthorized access, damage, or disruption — built on three pillars.

Confidentiality

Keep secrets secret

Only authorized people can view sensitive data. Enforced via encryption and access control.

Integrity

Keep data trustworthy

Information stays accurate and unaltered unless changed by an authorized action.

Availability

Keep systems usable

Authorized users can reliably access systems and data when needed.

05 / Malware

Malicious software types

Virus

Self-replicating code

Attaches to files and spreads when the host program runs.

Worm

Network-spreading

Replicates across networks without needing a host file or user action.

Ransomware

Holds data hostage

Encrypts files and demands payment for the decryption key.

Trojan

Disguised payload

Poses as legitimate software to deliver a hidden malicious function.

06 / Social Engineering

The human is the target

Most breaches don't start with code — they start with a convincing message that tricks a person into acting.

Phishing emails Spear phishing Pretexting Baiting Tailgating Vishing (voice)

DEFENSE: VERIFY THE REQUEST THROUGH A SECOND, TRUSTED CHANNEL BEFORE ACTING.

07 / Network Security

Guarding the perimeter

Firewall

Traffic filter

Allows or blocks traffic based on defined security rules.

VPN

Encrypted tunnel

Secures data in transit across untrusted networks.

Segmentation

Limit blast radius

Splits a network into zones so a breach in one doesn't reach all.

10 / Defense in Depth

Layered protection

Patch

Update systems

Apply security patches promptly to close known vulnerabilities.

Backup

3-2-1 rule

3 copies, 2 media types, 1 offsite — recover without paying ransom.

Train

Security awareness

Regularly teach staff and users to recognize threats.

Monitor

Detect early

Logging and alerting to catch incidents before they escalate.

11 / Lesson From the Field

A familiar pattern

Many large breaches share the same root cause: an unpatched system or a single phished credential gave an attacker an initial foothold, which then spread because the network lacked segmentation and monitoring.

TAKEAWAY: NO SINGLE CONTROL IS ENOUGH — LAYERS CATCH WHAT ONE GAP MISSES.

12 / Conclusion

Security is a habit, not a product

Tools matter, but consistent practice — patching, strong authentication, awareness, and monitoring — is what actually keeps systems safe.
